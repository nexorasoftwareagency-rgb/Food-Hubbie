# Complete Flow: Bot Promotions Module

## 1. Admin creates campaign
1. Admin opens Promotions tab in Dashboard
2. Composes message with tokens ({name}, {phone}, etc.)
3. Selects recipients (all customers / filtered / paste/upload CSV)
4. Optionally attaches image, enables coupon generation
5. Clicks "🚀 START CAMPAIGN"

## 2. Command pushed to Firebase
```
bot/{outlet}/commands/{cmdId}/
  action: "SEND_PROMOTION"
  campaignId: "PROMO-..."
  template: "Hello {name}!..."
  recipients: ["919876543210", ...]
  delayMs: 2000
  generateCoupons: false
  quietHours: { start: 22, end: 9 }
```

## 3. Bot picks up command
`initCommandListener()` receives the push, calls `runPromotionCampaign(sock, cmd)`.

## 4. Campaign loop (per recipient)
```
for each phone in recipients:
  1. Check killSwitch → abort if on
  2. Check quiet hours → sleep if active
  3. Check socket health → pause if dead
  4. Check cryptoErrorCount → pause if > 100
  5. Acquire lock → skip if another campaign running
  6. Check opt-out list → skip if opted out
  7. Check promotionalConsent → skip if false
  8. Personalize template with tokens
  9. Generate coupon code if enabled
  10. Send via sendPromotionalMessage() (no admin footer)
  11. Log result to promotions/logs/
  12. Persist progress every 10 sends
  13. Sleep (2s default, 30s every 50 sends)
```

## 5. Campaign completes
```
bot/{outlet}/promotions/campaigns/{id}/
  status: "done"
  completedAt: Date.now()
  totalSent: 140
  totalFailed: 2
```

## 6. Admin monitors live
- Progress bar: `currentIndex / recipients.length * 100%`
- Live log: phone, status (sent/failed/skipped), timestamp
- "🛑 KILL ALL" panic button

## 7. Bot restart mid-campaign
1. `startBot()` runs after reconnection
2. Scans `campaigns` for `status="running"`
3. Re-fetches the original command from `commands/`
4. Resumes from `currentIndex` stored in the campaign doc

## 8. Customer interaction
- Customer replies "STOP" → added to opt-out list, future campaigns skip them
- Customer replies "START" → re-opted-in, future campaigns include them
- Customer places order during campaign → normal order flow (no interference)

## 9. Cross-page flows
- **Discounts** — coupon codes generated by promotions can be redeemed in bot checkout
- **Orders** — campaign sends are logged in `discountsUsage` if coupons are generated
- **Analytics** — campaign performance visible in Admin Reports
