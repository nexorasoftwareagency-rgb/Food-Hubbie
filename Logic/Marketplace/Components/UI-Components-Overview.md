# UI Components — Overview

## Source
`src/components/ui/` — ~65 files generated by shadcn/ui CLI.

## Pattern
Each file wraps a Radix UI primitive with Tailwind CSS styling using `cn()` utility:
```tsx
// Example: button.tsx
const buttonVariants = cva(
  "inline-flex items-center justify-center rounded-md text-sm font-medium ...",
  { variants: { variant: { default: "bg-primary text-primary-foreground ..." }, ... } }
)
```

## Custom Components
| Component | Purpose |
|---|---|
| `SkeletonLoader` | Loading placeholder (types: banner, list, card) |
| `FloatingCart` | Fixed bottom cart bar with item count + total + checkout button |
| `empty.tsx` | Configurable empty state component |
| `spinner.tsx` | SVG loading spinner |
| `item.tsx` | Generic list item component |

## Decisions
- All shadcn/ui components are unmodified from generated defaults
- `cn()` utility uses `clsx` + `tailwind-merge` for class conflicts
- `cva` (class-variance-authority) for component variants
- Custom components (SkeletonLoader, FloatingCart) extend shadcn patterns
